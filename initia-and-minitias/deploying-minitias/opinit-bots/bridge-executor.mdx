---
title: Bridge Executor
---

The bridge executor monitors the L1 and L2 transactions and facilitate token bridging and withdrawals between the minitia and Initia L1 chain.

## Set Bridge Executor Environment Variables

<Note>
- `TYPEORM_DATABASE` should be the name of the database previously created
- `L2_CHAIN_ID` and `EXECUTOR_MNEMONIC` are your minitia's chain ID and bridge executor menmonic found in `$HOME/.minitia/artifacts/config.json`
- `BRIDGE_ID` can be found in `$HOME/.minitia/artifacts/artifacts.json`
</Note>

```sh .env.executor
TYPEORM_CONNECTION=postgres               # database connection (currently only support `postgres`)
TYPEORM_HOST=localhost                    # database host
TYPEORM_USERNAME=postgres                 # database username
TYPEORM_PASSWORD=postgres                 # database password
TYPEORM_DATABASE=minitia                   # database name
TYPEORM_PORT=5432                         # database port
TYPEORM_SYNCHRONIZE=true                  # synchronize database schema
TYPEORM_LOGGING=false                     # enable logging
TYPEORM_ENTITIES=dist/orm/*Entity.js      # entity path

L1_CHAIN_ID=initiation-2
L2_CHAIN_ID=local-minitia
L1_LCD_URI=https://lcd.initiation-2.initia.xyz
L1_RPC_URI=https://rpc.initiation-2.initia.xyz
L2_LCD_URI=http://127.0.0.1:1317
L2_RPC_URI=http://127.0.0.1:26657
BRIDGE_ID=1                                # need to be same as in $HOME/.minitia/artifacts/artifacts.json

EXECUTOR_PORT=5000
ENABLE_ORACLE=true
EXECUTOR_MNEMONIC='tent apple ...'         # need to be same as in $HOME/.minitia/artifacts/artifacts.json

# optional config
# EXECUTOR_L1_MONITOR_HEIGHT=0
# EXECUTOR_L2_MONITOR_HEIGHT=0
# SLACK_WEB_HOOK=''
```

| Name                        | Description                                   | Default                  |
|-----------------------------|-----------------------------------------------|--------------------------|
| L1_CHAIN_ID                 | Initia L1 Chain ID                            |                          |
| L2_CHAIN_ID                 | Your minitia's Chain ID                       |                          |
| L1_LCD_URI                  | L1 node LCD URI                               | [http://127.0.0.1:1317](http://127.0.0.1:1317)   |
| L1_RPC_URI                  | L1 node RPC URI                               | [http://127.0.0.1:26657](http://127.0.0.1:26657) |
| L2_LCD_URI                  | L2 node LCD URI                               | [http://127.0.0.1:1317](http://127.0.0.1:1317)   |
| L2_RPC_URI                  | L2 node RPC URI                               | [http://127.0.0.1:26657](http://127.0.0.1:26657) |
| BRIDGE_ID                   | Bridge ID                                     | ""                       |
| EXECUTOR_PORT               | Executor port                                 | 5000                     |
| EXECUTOR_MNEMONIC           | Mnemonic seed for executor                    | ""                       |
| SLACK_WEB_HOOK              | Slack web hook for notification (optional)    | ""                       |
| BATCH_SUBMITTER_ADDR        | Batch submitter address                       | ""                       |
| EXECUTOR_L1_MONITOR_HEIGHT  | L1 height to start monitoring from (optional) | 0                        |
| EXECUTOR_L2_MONITOR_HEIGHT  | L2 height to start monitoring from (optional) | 0                        |
| ENABLE_API_ONLY             | Enable API only mode (optional)               | false                    |
| ENABLE_ORACLE               | Enable Oracle (optional)                      | false                    |
| L1_CLIENT_ID                | L1 client id for Oracle (optional)            | ""                       |


## Run the Bridge Executor Bot

<CodeGroup>
```sh Node
npm run prod:executor
```
```sh Docker
docker run -d -p 5432:5432 --network host ghcr.io/initia-labs/opinit-bots:v0.1.16 prod:executor --env-file .env.executor
```
</CodeGroup>
