## Operator Address

The operator is the account you specify during VIP whitelisting; it receives the operator commission for your rollup.

Operator commission is a fixed percentage of the rollup's total VIP reward. For details, see [distribution to rollup operators](/home/core-concepts/vip/architecture#distribution-to-rollup-operators). If you need to replace the operator address itself, call `update_operator` with the bridge ID and the new operator address:

<Tab title="Updating Operator Address">
```bash
export BRIDGE_ID=29                                                      # Replace with your actual bridge ID. You can get it from `${REST_URL}/opinit/opchild/v1/bridge_info`
export NEW_OPERATOR_ADDRESS=init14zmga05vl324ddaw7wqwdqt9pcvjeeyae3jn2k  # Replace with the new operator address
export OPERATOR_KEY_NAME=operator                                        # Replace with your operator key. Check `initiad keys list` for the list of keys

# Mainnet (interwoven-1)
# VIP_CONTRACT='init182yxkv4gqfvz7tjyde6dfgjdr4ldqxklgmf23aju2u3cslnss7ys6dy6w8' 
# RPC_URL='https://rpc.initia.xyz'
# CHAIN_ID='interwoven-1'

# Testnet (initiation-2)
# VIP_CONTRACT='init1u4wvsgl0ksgma40w6fdv55nhy2d9f332kdmfqp0cdnzyhsxquk4swhuwfz'
# RPC_URL='https://rpc.testnet.initia.xyz'
# CHAIN_ID='initiation-2'

initiad tx move execute-json $VIP_CONTRACT vip update_operator --args '["'${BRIDGE_ID}'", "'${NEW_OPERATOR_ADDRESS}'"]' --from $OPERATOR_KEY_NAME --node $RPC_URL --chain-id $CHAIN_ID --gas-prices 0.015uinit --gas auto
```
</Tab>

## Operator Commission

Operator commissions can be adjusted, subject to the `operator_commission_max_rate` and `operator_commission_max_change_rate` settings defined during VIP whitelisting. 
To apply an update, the operator account must send an `update_operator_commission` message to the VIP contract that includes the bridge ID, the version (use 1 unless the rollup was previously delisted), and the new commission rate.

<Note>
When you whitelist your rollup on VIP, you need to set [`operator_commission_max_rate` and `operator_commission_max_change_rate`](https://github.com/initia-labs/vip-contract/blob/f8b9a7fef024ace362fcddbc09ef35d694e471db/sources/vip.move#L1176-L1177).

`operator_commission_max_rate` is the maximum commission rate that can be set for the operator. For example, if you set it to `0.1`, the operator commission can be at most 10%. This value is used to ensure that the operator commission does not exceed a certain threshold.

`operator_commission_max_change_rate` is the maximum rate at which the operator commission can change. For example, if you set it to `0.05`, the operator commission can increase or decrease by up to 5% at a time.
</Note>

<Tab title="Updating Operator Commission">
```bash
export BRIDGE_ID=29                         # Replace with your actual bridge ID. You can get it from `${REST_URL}/opinit/opchild/v1/bridge_info`
export VERSION=1                            # Use 1 unless your rollup has been delisted before
export NEW_COMMISSION_RATE=0.05             # Replace with the new commission rate (e.g., 0.05 for 5%)
export OPERATOR_KEY_NAME=operator           # Replace with your operator key name

# Mainnet (interwoven-1)
# VIP_CONTRACT='init182yxkv4gqfvz7tjyde6dfgjdr4ldqxklgmf23aju2u3cslnss7ys6dy6w8'
# RPC_URL='https://rpc.initia.xyz'
# CHAIN_ID='interwoven-1'

# Testnet (initiation-2)
# VIP_CONTRACT='init1u4wvsgl0ksgma40w6fdv55nhy2d9f332kdmfqp0cdnzyhsxquk4swhuwfz'
# RPC_URL='https://rpc.testnet.initia.xyz'
# CHAIN_ID='initiation-2'

initiad tx move execute-json $VIP_CONTRACT vip update_operator_commission --args '["'${BRIDGE_ID}'", "'${VERSION}'", "'${NEW_COMMISSION_RATE}'"]' --from $OPERATOR_KEY_NAME --node $RPC_URL --chain-id $CHAIN_ID --gas-prices 0.015uinit --gas auto
```
</Tab>
