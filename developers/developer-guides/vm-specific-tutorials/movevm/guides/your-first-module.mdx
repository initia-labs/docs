---
title: Your First Move Module
---

In this guide, we'll walk through the process of creating, compiling, and deploying a simple Move module using the Move Virtual Machine (MoveVM). We'll use the Initia L1 testnet for this example, but the process is similar for MiniMove rollups.

<Note>
Before you start, make sure you have set up the CLI for your network, created an account using the CLI, and that your account has some tokens to pay for gas. Follow the steps outlined in the [previous page](../setting-up) to get set up.
</Note>

<Steps>
<Step title="Setting Environment Variables">
Before we start, we need to set a few environment variables.

```bash
export RPC_URL=https://rpc.testnet.initia.xyz # Replace with the correct RPC URL for your network
export CHAIN_ID=initiation-2 # Replace with the correct chain ID for your network
export GAS_PRICES=0.015uinit # Replace with the correct gas price for your network
```

| Variable    | Description                                                                                                                |
|-------------|----------------------------------------------------------------------------------------------------------------------------|
| RPC_URL     | The RPC URL endpoint for the network you're connecting to. This is used to interact with the blockchain.  |
| CHAIN_ID    | The identifier for the specific blockchain network you're connecting to. In this case, it's the Initia testnet.            |
| GAS_PRICES  | The price of gas for transactions on the network. It's specified in the format of `{amount}{denom}`, where `denom` is the  |
|             | unique identifier for the token used to pay for gas.                                                                       |

For Initia L1 and Initia-hosted networks, you can find these values in the [networks](/resources/developer/initia-l1/) section of the documentation. For MiniMove rollups, you can find these values in the [Initia Registry](https://github.com/initia-labs/initia-registry/tree/main). repository
</Step>
<Step title="Clone the Example Repository">
First, clone the example repository and navigate to the `movevm` directory.

```bash
git clone https://github.com/initia-labs/examples.git
cd examples/movevm
```
</Step>
<Step title="Update Move.toml">
Next, before we can compiled our module, we need to update the `Move.toml` to set `example` to our deployer address.

To find your hex address, run the following command.

<CodeGroup>
```bash Initia L1
initiad keys parse $(initiad keys show example --address)
```
```bash Rollups
minitiad keys parse $(minitiad keys show example --address)
```
</CodeGroup>

You should get an output like this:

```bash
bytes: CC86AF3F776B95A7DED542035B3B666A9FDE2EE9
human: init
```

Copy the hex address and paste it into the `Move.toml` file as follows. Remember to append the "0x" prefix to the copied address.

```toml Move.toml
[package]
name = "movevm"
version = "0.0.0"

[dependencies]
InitiaStdlib = { git = "https://github.com/initia-labs/movevm.git", subdir = "precompile/modules/initia_stdlib", rev = "main" }

[addresses]
example =  "0xCC86AF3F776B95A7DED542035B3B666A9FDE2EE9" # Replace with your hex address copied from the previous step
```

</Step>
<Step title="Build the Module">
Once we have updated the `Move.toml` file, we can build our module.

<CodeGroup>
```bash Initia L1
initiad move build
```
```bash Rollups
minitiad move build
```
</CodeGroup>
</Step>
<Step title="Deploy the Module">
Once the compilation is complete, we can deploy our module.

<CodeGroup>
```bash Initia L1
initiad move deploy \
  --path . \
  --upgrade-policy COMPATIBLE \
  --from example \
  --gas auto --gas-adjustment 1.5 \
  --gas-prices $GAS_PRICES \
  --node $RPC_URL \
  --chain-id $CHAIN_ID
```
```bash Rollups
minitiad move deploy \
  --path . \
  --upgrade-policy COMPATIBLE \
  --from example \
  --gas auto --gas-adjustment 1.5 \
  --gas-prices $GAS_PRICES \
  --node $RPC_URL \
  --chain-id $CHAIN_ID
```
</CodeGroup>
</Step>
<Step title="Interact with the Module">
To test that our module is working correctly, we can try interacting with it. First, we will save a message to the module.

<CodeGroup>
```bash Initia L1
export MODULE_ADDRESS=0xcc86af3f776b95a7ded542035b3b666a9fde2ee9 # Replace with your deployer address
initiad tx move execute $MODULE_ADDRESS hello_world save_message \
  --args '["string:data"]'  \
  --node $RPC_URL \
  --from docs \
  --gas-prices $GAS_PRICES \
  --chain-id $CHAIN_ID
```
```bash Rollups
export MODULE_ADDRESS=0xcc86af3f776b95a7ded542035b3b666a9fde2ee9 # Replace with your deployer address
minitiad tx move execute $MODULE_ADDRESS hello_world save_message \
  --args '["string:data"]'  \
  --node $RPC_URL \
  --from docs \
  --gas-prices $GAS_PRICES \
  --chain-id $CHAIN_ID
```
</CodeGroup>

Once the transaction is successful, we can retrieve the message we just saved.

<CodeGroup>
```bash Initia L1
initiad query move view $MODULE_ADDRESS hello_world view_message \
    --args '["address:0xcc86af3f776b95a7ded542035b3b666a9fde2ee9"]' \
    --node $RPC_URL
```
```bash Rollups
minitiad query move view $MODULE_ADDRESS hello_world view_message \
    --args '["address:0xcc86af3f776b95a7ded542035b3b666a9fde2ee9"]' \
    --node $RPC_URL
```
</CodeGroup>

If both steps are successful, you should see the following output:

```
data: '"data"'
events: []
gas_used: "1071"
```
</Step>
</Steps>
